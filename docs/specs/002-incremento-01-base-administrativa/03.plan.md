# Spec: 002 - Incremento 01: Base Administrativa

## 3. Plan de Ejecución

### Objetivo del Incremento

Crear la estructura fundamental del proyecto Django, los modelos de datos para las entidades administrativas principales, un entorno de desarrollo dockerizado y la capacidad de registrar y visualizar estos datos a través del panel de administración de Django.

### Tareas Detalladas

El trabajo se dividirá en las siguientes tareas secuenciales:

1.  **Configuración del Entorno de Desarrollo:**
    -   **Tarea 1.1:** Crear el archivo `docker-compose.yml` en la raíz del proyecto para definir el servicio de la base de datos PostgreSQL.
        -   **Sub-tareas:**
            -   Especificar la imagen `postgres:latest`.
            -   Definir un volumen nombrado para la persistencia de datos.
            -   Configurar las variables de entorno para `POSTGRES_DB`, `POSTGRES_USER` y `POSTGRES_PASSWORD`.
    -   **Tarea 1.2:** Crear un archivo `.gitignore` para excluir archivos y directorios innecesarios del control de versiones (ej. `__pycache__/`, `*.pyc`, `mediafiles/`, `venv/`).
    -   **Tarea 1.3:** Crear el entorno virtual de Python (`python -m venv venv`) y activarlo.
    -   **Tarea 1.4:** Crear un archivo `requirements.txt` y añadir las dependencias iniciales: `django`, `psycopg2-binary`.
    -   **Tarea 1.5:** Instalar las dependencias (`pip install -r requirements.txt`).

2.  **Inicialización y Configuración del Proyecto Django:**
    -   **Tarea 2.1:** Inicializar el proyecto Django con `django-admin startproject footballer .`.
    -   **Tarea 2.2:** Crear la aplicación principal con `python manage.py startapp core`.
    -   **Tarea 2.3:** Modificar `footballer/settings.py`:
        -   Añadir `core` a la lista de `INSTALLED_APPS`.
        -   Configurar la sección `DATABASES` para que se conecte al contenedor de Docker, leyendo las credenciales desde las variables de entorno.
        -   Definir `MEDIA_URL` y `MEDIA_ROOT` para la gestión de archivos.
    -   **Tarea 2.4:** Modificar `footballer/urls.py` para servir los archivos de medios durante el desarrollo.

3.  **Implementación de Modelos y Migración:**
    -   **Tarea 3.1:** Definir el modelo abstracto `BaseModel` en `core/models.py`.
    -   **Tarea 3.2:** Implementar los modelos `Federacion`, `Liga`, `Sede`, `Equipo`, `Temporada` y `Jugador` en `core/models.py`, heredando de `BaseModel`.
    -   **Tarea 3.3:** Generar el primer archivo de migración con `python manage.py makemigrations`.
    -   **Tarea 3.4:** Levantar el contenedor de la base de datos con `docker-compose up -d`.
    -   **Tarea 3.5:** Aplicar la migración a la base de datos con `python manage.py migrate`.

4.  **Configuración del Panel de Administración:**
    -   **Tarea 4.1:** Registrar todos los modelos creados en `core/admin.py` para que sean gestionables a través del `Django Admin`.
    -   **Tarea 4.2:** Crear un superusuario para acceder al panel de administración con `python manage.py createsuperuser`.
    -   **Tarea 4.3:** Verificar que es posible acceder al `Django Admin` y realizar operaciones CRUD en todas las entidades.

5.  **Creación del Comando de Seeding:**
    -   **Tarea 5.1:** Crear la estructura de directorios `core/management/commands/`.
    -   **Tarea 5.2:** Implementar el script `seed_data.py` dentro de ese directorio. El script deberá crear un conjunto de datos inicial coherente para todas las entidades.
    -   **Tarea 5.3:** Ejecutar el comando `python manage.py seed_data` y verificar que la base de datos se puebla correctamente.

### Verificación y Cierre

-   Al finalizar todas las tareas, se debe verificar que se cumplen todos los criterios de aceptación definidos en el archivo `01.requirements.md`.
-   El código debe estar en un estado funcional y listo para servir como base para el siguiente incremento.
-   Se creará el archivo `04.exec.phase-01.md` para documentar la ejecución y los resultados de cada tarea.
