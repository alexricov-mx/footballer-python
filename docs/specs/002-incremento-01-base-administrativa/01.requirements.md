# Spec: 002 - Incremento 01: Base Administrativa

## 1. Requisitos

### Resumen

Este documento define los requisitos para la primera fase de desarrollo del proyecto "Footballer". El objetivo de este incremento es establecer la estructura fundamental de la aplicación, incluyendo los modelos de datos para las entidades administrativas principales y la configuración inicial del proyecto.

### Requisitos Funcionales (Extraídos de la Épica)

El alcance de este incremento se centra en las siguientes `Features` definidas en `Epica-Features-Footballer-06.md`:

-   **F01. Gestión de federaciones:**
    -   Crear, editar, habilitar/deshabilitar y consultar federaciones.
    -   Asociar ligas a una federación.
    -   Validar que el nombre de la federación sea único y obligatorio.

-   **F02. Gestión de ligas:**
    -   Crear, editar, habilitar/deshabilitar y consultar ligas.
    -   Asociar una liga a una federación.
    -   Validar que una liga pertenezca a una federación y que su nombre sea único dentro de esa federación.

-   **F03. Gestión de temporadas:**
    -   Crear, editar y habilitar/deshabilitar temporadas asociadas a una liga.
    -   Definir nombre, modalidad de calendario (una o doble vuelta) y estado.
    -   Bloquear modificaciones una vez que la temporada ha iniciado.

-   **F04. Gestión de equipos:**
    -   Crear, editar, habilitar/deshabilitar y consultar equipos.
    -   El modelo debe estar preparado para que un equipo pueda asociarse a múltiples ligas y temporadas en incrementos futuros.

-   **F05. Gestión de jugadores:**
    -   Crear, editar, habilitar/deshabilitar y consultar jugadores.
    -   El modelo debe estar preparado para asociar un jugador a un equipo.

-   **F07. Gestión de sedes:**
    -   Crear, editar, habilitar/deshabilitar y consultar sedes.
    -   Campos requeridos: Nombre, dirección, ciudad, referencia.
    -   Campo opcional: Foto.
    -   El sistema debe proveer una imagen genérica si no se sube una foto.

### Criterios de Aceptación del Incremento

Al finalizar este incremento, se deberá cumplir lo siguiente:

1.  **Modelos de Datos:** Deben existir los modelos Django para `Federacion`, `Liga`, `Temporada`, `Equipo`, `Jugador` y `Sede`, reflejando las relaciones y campos definidos en la épica.
2.  **Migraciones:** Se debe haber generado y aplicado al menos una migración inicial que cree las tablas correspondientes en la base de datos.
3.  **Administración Básica:** Debe ser posible realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para cada una de estas entidades, preferiblemente a través del panel de administración de Django (`Django Admin`) para acelerar el desarrollo.
4.  **Configuración de Medios:** El sistema debe estar configurado para manejar la carga de la foto de la sede en el sistema de archivos local y mostrar una imagen por defecto si no existe.
5.  **Entorno Reproducible:** La configuración de la base de datos PostgreSQL debe estar gestionada a través de Docker para garantizar consistencia.
6.  **Datos Iniciales:** Debe existir un `management command` que permita poblar la base de datos con datos de prueba para facilitar el desarrollo y las pruebas de los siguientes incrementos.

### Requerimientos No Funcionales

-   **Stack Tecnológico:** Python, Django, PostgreSQL.
-   **Entorno de Desarrollo:** La base de datos PostgreSQL se ejecutará en un contenedor de Docker.
-   **Gestión de Archivos:** Las fotos de las sedes se almacenarán en el sistema de archivos local.
-   **Población de Datos:** Se creará un `management command` de Django para el "seeding" de datos.
