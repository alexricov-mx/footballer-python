# Spec: 004 - Incremento 03: Gestión de Calendarios y Partidos

## 3. Plan de Implementación

### 3.1. Fases y Tareas

El trabajo se dividirá en las siguientes fases, ejecutadas en secuencia.

---

### **Fase 1: Modelos y Base de Datos**

*   **Objetivo:** Definir la estructura de datos para árbitros, sedes, jornadas y partidos.
*   **Duración Estimada:** 1 hora.

| Tarea ID | Descripción                                                                                             | Archivos Afectados                               |
| :------- | :------------------------------------------------------------------------------------------------------ | :----------------------------------------------- |
| T.3.1.1  | **Crear el modelo `Arbitro`**: Añadir la clase `Arbitro` en `core/models.py` con sus campos.              | `core/models.py`                                 |
| T.3.1.2  | **Crear el modelo `Sede`**: Añadir la clase `Sede` en `core/models.py` con sus campos.                    | `core/models.py`                                 |
| T.3.1.3  | **Crear el modelo `Jornada`**: Añadir la clase `Jornada` y su relación con `Temporada`.                   | `core/models.py`                                 |
| T.3.1.4  | **Crear el modelo `Partido`**: Añadir la clase `Partido` con sus relaciones y campos de estado.           | `core/models.py`                                 |
| T.3.1.5  | **Generar Migraciones**: Ejecutar `makemigrations` para crear los archivos de migración para los nuevos modelos. | `core/migrations/`                               |
| T.3.1.6  | **Aplicar Migraciones**: Ejecutar `migrate` para actualizar el esquema de la base de datos.               | - (Comando de terminal)                          |

---

### **Fase 2: Integración en el Panel de Administración**

*   **Objetivo:** Hacer que los nuevos modelos sean gestionables a través del panel de administración de Django.
*   **Duración Estimada:** 1 hora.

| Tarea ID | Descripción                                                                                             | Archivos Afectados |
| :------- | :------------------------------------------------------------------------------------------------------ | :----------------- |
| T.3.2.1  | **Registrar Modelos**: Importar y registrar `Arbitro`, `Sede`, `Jornada` y `Partido` en `core/admin.py`. | `core/admin.py`    |
| T.3.2.2  | **Configurar `PartidoAdmin`**: Personalizar la vista de lista (`list_display`, `list_filter`) para `Partido`. | `core/admin.py`    |
| T.3.2.3  | **(Opcional) Configurar `JornadaAdmin` con `PartidoInline`**: Crear una vista anidada para ver los partidos de una jornada. | `core/admin.py`    |

---

### **Fase 3: Actualización de Datos de Prueba**

*   **Objetivo:** Poblar la base de datos con datos de prueba coherentes para las nuevas entidades.
*   **Duración Estimada:** 1.5 horas.

| Tarea ID | Descripción                                                                                             | Archivos Afectados                     |
| :------- | :------------------------------------------------------------------------------------------------------ | :------------------------------------- |
| T.3.3.1  | **Crear Árbitros y Sedes**: En `seed_data`, añadir lógica para crear una lista de árbitros y sedes de prueba. | `core/management/commands/seed_data.py` |
| T.3.3.2  | **Crear Jornadas**: Añadir lógica para crear 17 jornadas para la temporada "Apertura 2024".               | `core/management/commands/seed_data.py` |
| T.3.3.3  | **Crear Partidos**: Añadir lógica para generar 9 partidos por jornada, asignando equipos, sedes y árbitros. | `core/management/commands/seed_data.py` |
| T.3.3.4  | **Ejecutar y Validar Seeder**: Correr el comando `seed_data` para asegurar que se ejecuta sin errores.      | - (Comando de terminal)                |

---

### **Fase 4: Verificación y Cierre**

*   **Objetivo:** Realizar una validación final y documentar el cierre del incremento.
*   **Duración Estimada:** 0.5 horas.

| Tarea ID | Descripción                                                                                             | Archivos Afectados                               |
| :------- | :------------------------------------------------------------------------------------------------------ | :----------------------------------------------- |
| T.3.4.1  | **Verificación Manual**: Acceder al panel de admin y confirmar que los árbitros, sedes, jornadas y partidos se han creado y se muestran correctamente. | - (Navegador web)                                |
| T.3.4.2  | **Crear Resumen del Incremento**: Documentar los resultados y conclusiones en el archivo `05.summary.md`.  | `docs/specs/004.../05.summary.md`                |

### 3.2. Bitácora de Ejecución

Se creará un archivo `04.exec.phase-01.md` para registrar el progreso y los resultados de cada tarea durante la fase de ejecución.
