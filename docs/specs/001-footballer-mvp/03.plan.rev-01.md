# Spec: 001 - Footballer MVP

## 3. Plan de Ejecución - Revisión 01

Este documento refina el plan de ejecución original (`03.plan.md`) incorporando decisiones técnicas clave para la configuración del entorno de desarrollo.

### Resumen de Decisiones (Apéndice al Plan Original)

1.  **Base de Datos (PostgreSQL):**
    -   **Decisión:** Se utilizará **Docker** para gestionar la instancia de PostgreSQL.
    -   **Acción:** Se creará un archivo `docker-compose.yml` en la raíz del proyecto para definir el servicio de la base de datos. Esto asegura un entorno de desarrollo consistente y reproducible.
    -   **Impacto en el Plan:** La Tarea 1 del Incremento 1 (`Inicializar proyecto Django`) ahora incluye la creación y configuración del `docker-compose.yml`.

2.  **Gestión de Archivos (Fotos de Sedes):**
    -   **Decisión:** Durante el desarrollo, los archivos se gestionarán en el **sistema de archivos local**.
    -   **Acción:** Se configurarán las variables `MEDIA_URL` y `MEDIA_ROOT` en `settings.py` para que Django guarde y sirva las imágenes desde un directorio local (ej. `/mediafiles/`).
    -   **Impacto en el Plan:** La Tarea 3 del Incremento 1 (`Definir los modelos`) se mantiene, pero la implementación de las vistas que manejen la subida de archivos en el futuro considerará esta configuración.

3.  **Población de Datos de Prueba (Seeding):**
    -   **Decisión:** Se creará un **`management command` de Django** para la población de datos.
    -   **Acción:** Se añadirá una nueva tarea al final del Incremento 1 para crear un comando (ej. `python manage.py seed_data`) que pueble la base de datos con un conjunto de datos inicial (federaciones, ligas, equipos, etc.).
    -   **Impacto en el Plan:** Se añade una tarea explícita de "Crear management command para seeding" al Incremento 1.

### Plan de Ejecución Ajustado (Incremento 1)

A continuación, se muestra el plan detallado y ajustado para el primer incremento, reflejando las decisiones tomadas.

#### Incremento 1: Base Administrativa (Revisado)

**Objetivo:** Crear la estructura fundamental del proyecto Django, los modelos de datos para las entidades principales y un entorno de desarrollo dockerizado.

-   **Tareas de Infraestructura y Backend:**
    1.  **Crear `docker-compose.yml`:** Definir un servicio `db` para PostgreSQL, especificando imagen, volumen para persistencia de datos y variables de entorno (usuario, contraseña, nombre de la base de datos).
    2.  **Inicializar proyecto Django:** Ejecutar `django-admin startproject footballer .` y `python manage.py startapp core`.
    3.  **Ajustar `settings.py`:**
        -   Configurar la conexión a la base de datos para que apunte al contenedor de Docker.
        -   Registrar la app `core`.
        -   Configurar `MEDIA_URL` y `MEDIA_ROOT` para el manejo de archivos locales.
    4.  **Definir los modelos en `core/models.py`:** Crear las clases para `Federacion`, `Liga`, `Temporada`, `Equipo`, `Jugador` y `Sede`, utilizando un `BaseModel` abstracto para campos comunes.
    5.  **Generar y aplicar la primera migración:** Ejecutar `makemigrations` y `migrate`.
    6.  **Crear `management command` para seeding:**
        -   Crear la estructura de directorios `core/management/commands/`.
        -   Implementar el comando `seed_data.py` para crear un conjunto de datos inicial coherente.

-   **Tareas Frontend:**
    1.  Configurar el directorio de plantillas y crear `templates/base.html` con la integración de Bootstrap desde un CDN.

-   **Pruebas:**
    1.  Configurar el framework de pruebas de Django.
    2.  Crear pruebas unitarias para la creación de cada modelo.
    3.  Probar que el comando `seed_data` se ejecuta correctamente y puebla la base de datos.

*(Los incrementos 2 al 7 se mantienen como en el plan original, pero se ejecutarán sobre la base establecida en este incremento revisado).*
