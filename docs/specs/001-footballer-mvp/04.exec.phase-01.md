# Spec: 001 - Footballer MVP

## 4. Ejecución - Fase 1

### 4.1. Objetivo de la Fase

Esta fase corresponde al **Incremento 1: Base Administrativa**. El objetivo es establecer la estructura fundamental del proyecto Django, crear los modelos de datos para las entidades principales y configurar la base del frontend.

### 4.2. Log de Ejecución

Aquí se documentarán los pasos, decisiones y resultados de la implementación.

#### Paso 1: Inicialización del Proyecto Django

-   **Acción:** Crear la estructura del proyecto y la primera aplicación.
-   **Comandos:**
    ```bash
    django-admin startproject footballer .
    python manage.py startapp core
    ```
-   **Resultado:** Proyecto `footballer` y app `core` creados. Se ajustará `settings.py` para registrar la nueva app.

#### Paso 2: Definición de Modelos de Datos

-   **Acción:** Traducir las entidades del análisis a modelos de Django en `core/models.py`.
-   **Código Propuesto:**
    ```python
    # core/models.py

    from django.db import models

    class BaseModel(models.Model):
        """Modelo base con campos de auditoría y estado."""
        created_at = models.DateTimeField(auto_now_add=True, verbose_name="Fecha de creación")
        updated_at = models.DateTimeField(auto_now=True, verbose_name="Fecha de actualización")
        is_active = models.BooleanField(default=True, verbose_name="Activo")

        class Meta:
            abstract = True

    class Federacion(BaseModel):
        nombre = models.CharField(max_length=100, unique=True)

        def __str__(self):
            return self.nombre

    class Liga(BaseModel):
        federacion = models.ForeignKey(Federacion, on_delete=models.CASCADE)
        nombre = models.CharField(max_length=100)

        class Meta:
            unique_together = ('federacion', 'nombre')

        def __str__(self):
            return f"{self.nombre} ({self.federacion.nombre})"

    class Temporada(BaseModel):
        # ... (se completarán los demás modelos)
    
    class Equipo(BaseModel):
        # ...

    class Jugador(BaseModel):
        # ...

    class Sede(BaseModel):
        # ...
    ```
-   **Decisión:** Se crea una `BaseModel` abstracta para reutilizar los campos de auditoría (`created_at`, `updated_at`) y el estado (`is_active`) en todas las entidades principales, promoviendo código DRY (Don't Repeat Yourself).

#### Paso 3: Migración de la Base de Datos

-   **Acción:** Generar el script de migración y aplicarlo a la base de datos PostgreSQL.
-   **Comandos:**
    ```bash
    python manage.py makemigrations core
    python manage.py migrate
    ```
-   **Resultado:** El esquema de la base de datos se creará reflejando los modelos definidos.

#### Paso 4: Configuración del Frontend

-   **Acción:** Configurar la estructura de plantillas y añadir Bootstrap.
-   **Tareas:**
    1.  Crear el directorio `templates` en la raíz del proyecto.
    2.  Ajustar `settings.py` para que Django reconozca este directorio.
    3.  Crear `templates/base.html` y vincular el CDN de Bootstrap 5.

### 4.3. Verificación

Al final de esta fase, se debe verificar que:
-   [ ] El proyecto Django se ejecuta sin errores.
-   [ ] Las tablas de las entidades base existen en la base de datos PostgreSQL.
-   [ ] Se puede acceder al panel de administración de Django (`/admin`).
-   [ ] La plantilla base con Bootstrap se renderiza correctamente en una vista de prueba.
