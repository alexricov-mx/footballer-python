# Spec: 001 - Footballer MVP

## 3. Plan de Ejecución

Este plan detalla las tareas necesarias para desarrollar el MVP de Footballer, divididas en 7 incrementos funcionales. Cada incremento representa un conjunto de funcionalidades que pueden ser desarrolladas, probadas y entregadas de manera cohesiva.

### Definición de "Hecho" (Definition of Done) para el MVP

-   **Código Completo:** Todas las features del MVP están implementadas en Python/Django.
-   **Base de Datos Migrada:** El esquema de PostgreSQL está actualizado con todas las migraciones de Django.
-   **Pruebas Unitarias:** Las lógicas de negocio críticas (generación de calendario, cargas masivas, cálculo de tablas) tienen pruebas unitarias.
-   **Interfaz Funcional:** Todas las vistas están implementadas con Bootstrap y son funcionales en los navegadores web modernos.
-   **Roles y Permisos Aplicados:** La seguridad a nivel de vistas y lógica de negocio está implementada y probada para todos los roles.
-   **Documentación de Spec Completa:** Todos los documentos de la spec (01 a 05) están completos y reflejan el estado final del MVP.
-   **Sin Regresiones Críticas:** No existen bugs bloqueantes conocidos en las funcionalidades del MVP.

---

### Incremento 1: Base Administrativa

**Objetivo:** Crear la estructura fundamental del proyecto Django y los modelos de datos para las entidades principales.

-   **Tareas Backend:**
    1.  Inicializar proyecto Django (`django-admin startproject footballer`).
    2.  Crear una app de Django llamada `core` (`python manage.py startapp core`).
    3.  Definir los modelos en `core/models.py` para: `Federacion`, `Liga`, `Temporada`, `Equipo`, `Jugador`, `Sede`.
    4.  Incluir campos de auditoría básicos (`created_at`, `updated_at`) y estado (`is_active`).
    5.  Generar y aplicar la primera migración (`makemigrations`, `migrate`).
-   **Tareas Frontend:**
    1.  Configurar plantillas de Django y la integración de Bootstrap.
    2.  Crear una plantilla base (`base.html`) con la estructura de navegación principal.
-   **Pruebas:**
    1.  Configurar el framework de pruebas de Django.
    2.  Crear pruebas unitarias para la creación básica de cada modelo.

### Incremento 2: Participación y Plantillas

**Objetivo:** Modelar y construir la lógica para que los equipos participen en ligas y temporadas.

-   **Tareas Backend:**
    1.  Crear los modelos intermedios `LigaEquipo` y `TemporadaEquipo` para las relaciones N:M.
    2.  Implementar vistas (CBV) para asociar/desasociar equipos de una liga.
    3.  Implementar vistas para definir la plantilla de equipos de una temporada.
    4.  Aplicar migraciones.
-   **Tareas Frontend:**
    1.  Crear plantillas de Django para gestionar la participación de equipos en ligas y temporadas.
-   **Pruebas:**
    1.  Probar que un equipo puede estar en múltiples ligas.
    2.  Probar que la plantilla de una temporada es independiente.

### Incremento 3: Árbitros y Permisos

**Objetivo:** Establecer el sistema de roles y permisos.

-   **Tareas Backend:**
    1.  Crear el modelo `Arbitro` con el campo booleano `es_arbitro_general`.
    2.  Crear los Grupos de Django: `AdminGeneral`, `AdminLiga`, `Arbitro`.
    3.  Implementar vistas CRUD para `Arbitro`.
    4.  Crear `mixins` o decoradores para proteger las vistas según el rol y el alcance.
-   **Tareas Frontend:**
    1.  Ajustar las plantillas para mostrar/ocultar opciones según los permisos del usuario.
-   **Pruebas:**
    1.  Probar que un `AdminLiga` no puede acceder a funciones de `AdminGeneral`.
    2.  Probar las restricciones de acceso a las vistas protegidas.

### Incremento 4: Calendario

**Objetivo:** Implementar la generación y carga de calendarios.

-   **Tareas Backend:**
    1.  Crear los modelos `Jornada` y `Partido`.
    2.  Implementar el servicio de generación de calendario (algoritmo round-robin).
    3.  Implementar la vista para la carga de calendario desde Excel, incluyendo:
        -   Servicio de lectura y validación del archivo (`pandas`).
        -   Lógica de previsualización.
        -   Carga transaccional a la base de datos.
-   **Tareas Frontend:**
    1.  Crear la interfaz para iniciar la generación automática.
    2.  Crear el formulario para la subida del archivo Excel y la visualización de la previsualización y errores.
-   **Pruebas:**
    1.  Probar la generación de calendario con número par e impar de equipos.
    2.  Probar la validación de un archivo Excel con errores.

### Incremento 5: Operación de Partidos

**Objetivo:** Permitir la captura de resultados y eventos de los partidos.

-   **Tareas Backend:**
    1.  Crear el modelo `EventoPartido`.
    2.  Crear la vista de captura de resultado, usando `formsets` para los eventos.
    3.  Implementar la lógica transaccional para el cierre del partido.
    4.  Implementar las vistas para reagendar partidos.
-   **Tareas Frontend:**
    1.  Diseñar la plantilla para la captura de resultados, permitiendo añadir/quitar eventos dinámicamente con JavaScript.
-   **Pruebas:**
    1.  Probar que el marcador final debe coincidir con los eventos de gol.
    2.  Probar que un partido no se puede cerrar sin goleadores si el marcador es > 0.

### Incremento 6: Consultas y Tablas

**Objetivo:** Mostrar el progreso y las estadísticas de la temporada.

-   **Tareas Backend:**
    1.  Implementar el servicio o `Manager` para calcular la Tabla General usando el ORM.
    2.  Implementar el servicio o `Manager` para calcular la Tabla de Goleadores.
    3.  Crear las vistas (`ListView`) para mostrar ambas tablas, pasando los datos calculados al contexto.
-   **Tareas Frontend:**
    1.  Crear las plantillas para la Tabla General y la Tabla de Goleadores.
    2.  Crear la vista de jornadas para ver el progreso de la temporada.
-   **Pruebas:**
    1.  Probar los criterios de ordenamiento de la tabla general.
    2.  Probar el ranking compartido en la tabla de goleadores.

### Incremento 7: Cargas y Exportaciones

**Objetivo:** Finalizar las funcionalidades de importación y exportación de datos.

-   **Tareas Backend:**
    1.  Implementar las vistas y servicios para la carga masiva de `Equipos` y `Jugadores`.
    2.  Implementar la funcionalidad de exportación a Excel para todas las tablas principales (`pandas` o `XlsxWriter`).
-   **Tareas Frontend:**
    1.  Añadir botones de "Exportar a Excel" en las vistas correspondientes.
    2.  Crear las interfaces para la carga de equipos y jugadores.
-   **Pruebas:**
    1.  Probar la validación de los layouts de carga de equipos y jugadores.
    2.  Verificar que los archivos Excel exportados contengan los datos correctos.
